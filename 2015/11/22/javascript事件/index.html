javascript事件

JavaScript与HTML之间的交互式通过事件实现的。

###事件流###

  1. 概念
    事件流描述的是从页面中接收事件的顺序
  2. 过程
    2.1 事件冒泡和事件捕获

    如图，事件冒泡是IE事件流，即事件开始时由最具体的元素接收，然后逐级向上传播较为不具体的节点(document)。而事件捕获由NetScape提出，恰好相反。
    一个简单的例子，就是一张纸上画着好几个同心圆，你用手指指向圆心处，你所在指向的圆心是都是那些圆的圆心。

“DOM2级事件”规定的事件流包括三个阶段：事件捕获阶段、处于目标阶段、事件冒泡阶段。
2.2 阻止事件继续传递的方法

  • event.stopPropagation()
  • 删除相应元素，因为目标元素在文档中是事件冒泡的前提

###事件处理程序###

  1. 概念
    事件是用户或者浏览器自身执行的某种动作，如click,load,mouseover等，而去响应这些事件的函数称之为事件处理程序(事件侦听器),事件处理程序的名字以“on”开头，如onclick,onload等

  2. 有哪几种事件处理程序类型
    2.1 HTML事件处理程序

    <div onclick="alert('hello world')">hello world</div>
<div onclick="hi()">simple life</div>

    如这样直接写在HTML代码中的
    2.2 DOM0级事件处理程序
    通过JavaScript指定方法处理程序的传统方式，将一个函数赋值给一个事件处理程序属性。简单，跨浏览器。

这个也比较常见
2.3 DOM2级事件处理程序
addEventListener/removeEventListener
对于IE，attachEvent/detachEvent

注意两点1 第三个参数false,表示事件会在冒泡阶段触发；为true则表示事件会在捕获阶段触发，这个和我们什么时候阻止以及事件委托有关系。2 上述removeEventListener事件没起作用因为两者的function不一样，所以要将这个function单独写出来。

我们在做兼容性处理时候，可以重新定义一个函数做判断兼容上述情况的事件处理程序

###事件对象Event###

  1. 概念
    在触发DOM上某个事件时，会产生一个事件对象event，这个对象包含着所有与事件有关的信息。
  2. DOM中和IE中的事件对象
    参见w3cschool-DOM Event对象
    常见的几个注意下
    • currentTarget 表示其事件处理程序当前正在处理事件的那个元素
    • target 表示事件的目标
    • preventDefault() 取消事件的默认行为(cancelable = true)
    • stopPropagation() 取消事件进一步捕获或者冒泡(bubbles=true)

      注意：e.target e.currentTarget this在哪些情况下是一样的
      对于IE中的事件对象

    • cancelBubble 取消事件冒泡 相当于DOM中的stopPropagation()
    • returnValue 取消事件默认行为(默认true 设置false取消默认行为)
    • srcElement 事件目标 与DOM中的target一样

      IE中在使用DOM0级方法处理程序时，event对象作为window对象的一个属性存在

###内存和性能###
1 为什么
在JavaScript中，添加到页面上的事件处理程序数量将直接影响页面整体运行性能。原因有：

  1. 每个函数都是对象，都会占内存；内存中对象越多，性能就越差；
  2. DOM的访问次数
